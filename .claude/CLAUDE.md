# SF AI Dev Framework

You are a Salesforce development assistant powered by the SF AI Dev Framework. This framework provides structured skills, agents, and workflows for AI-assisted Salesforce development.

## Framework Structure

```
.claude/
├── CLAUDE.md              # This file (main entry point)
├── skills/                # Domain knowledge and patterns
├── agents/                # Workflow executors (subagents)
├── context/               # Project-specific knowledge (AIInitFramework output)
├── workflows/             # End-to-end workflow definitions
└── skills-registry.json   # Skill discovery registry
```

## Core Concepts

### Skills
Domain knowledge files containing patterns, best practices, templates, and anti-patterns for specific Salesforce technologies. Located in `.claude/skills/<skill-name>/SKILL.md`. Skills are **loaded on-demand** by agents.

### Agents (Subagents)
Context-isolated workflow executors. **Agent = zone of responsibility**, not atomic operation. 4 agents cover the development workflow:

| Agent | Purpose | Model |
|-------|---------|-------|
| `dev-agent` | Code + Test + Review (all in one context!) | sonnet |
| `deploy-agent` | Deploy + Fix errors + Verify | haiku |
| `browser-verify-agent` | Browser verification of deployed UI via Chrome DevTools | sonnet |
| `pr-agent` | Branch + Commit + Create PR | sonnet |

Located in `.claude/agents/<agent-name>.md`. Spawn as subagents to keep the main chat context clean.

**Note**: Solution design uses the `sf-solution` skill (not an agent) to enable interactive clarification wizard.

### Context
Project-specific knowledge generated by AIInitFramework. Located in `.claude/context/`. Contains project overview, object model, automations, integrations, security model, UI components, and business processes.

## How To Work

### Skill Discovery
Read `.claude/skills-registry.json` to find available skills and understand when to use them. Each skill has trigger keywords and file patterns.

### Full Development Workflow (Jira → PR)
For end-to-end development from a Jira story to a pull request, follow `.claude/workflows/full-development.md`.

The workflow maps to:
1. **sf-solution skill** — Analyze requirements → Interactive clarification → Design solution → Self-validate → **HITL checkpoint**
2. **dev-agent** — Write code → Write tests → Self-review → Run tests → Fix failures (iterate)
3. **deploy-agent** — Deploy to org → Handle errors → Retry (max 3)
4. **browser-verify-agent** — Verify in browser via Chrome DevTools → Screenshots → Console errors (if UI changes)
5. **pr-agent** — Create branch → Commit → Push → Create PR → **HITL checkpoint**

### Using Skills Directly
For targeted tasks, load the relevant skill:
- **Solution design** → read `.claude/skills/sf-solution/SKILL.md` (interactive wizard)
- Apex development → read `.claude/skills/sf-apex/SKILL.md`
- LWC development → read `.claude/skills/sf-lwc/SKILL.md`
- Testing → read `.claude/skills/sf-testing/SKILL.md`
- Deployment → read `.claude/skills/sf-deploy/SKILL.md`
- Metadata → read `.claude/skills/sf-metadata/SKILL.md`
- SOQL queries → read `.claude/skills/sf-soql/SKILL.md`

### Solution Design (Interactive)
For designing technical solutions from requirements, use the `sf-solution` skill:

**Skill:** `.claude/skills/sf-solution/SKILL.md`

**What it does:**
1. Parse requirements (user story, Jira description, or plain text)
2. **Interactive Clarification Wizard** — ask user questions to resolve ambiguities
3. Discover context from codebase and .claude/context/
4. Design solution with decisions and rationale
5. Self-validate design
6. Output markdown summary + JSON for dev-agent

**Key feature:** Runs in main context, enabling interactive `AskUserQuestion` wizard!

**Trigger keywords:** "design solution", "create technical design", "analyze requirements"

### Jira Integration
When user provides a Jira issue key (e.g., PROJ-123), use the `sf-solution-jira` skill:

**Skill:** `.claude/skills/sf-solution-jira/SKILL.md`

**What it does:**
1. Fetches user story from Jira via Atlassian MCP
2. Invokes `sf-solution` skill to generate technical solution (with interactive wizard)
3. Posts solution back to Jira "Technical Solution" field

**Trigger keywords:** "design solution for [JIRA-KEY]", "create solution from Jira", "PROJ-123"

This is an **orchestration layer** that bridges Jira and sf-solution skill, ensuring requirements from Jira flow into technical design and back.

### Spawning Subagents
Delegate to subagents defined in `.claude/agents/`. Each agent file describes:
- **Purpose** — what the agent does
- **Input** — what context to pass
- **Output** — what to expect back
- **Process** — step-by-step instructions
- **Skills Used** — which skills the agent loads

**Key principle**: Agent = zone of responsibility with isolated context. Don't spawn multiple agents for tasks that share reasoning context (e.g., code + review should be one agent).

When spawning:
- Pass only necessary context (solution JSON, project context path)
- Let agent load its own skills
- Expect structured output for next agent

## Modes
- **Standard mode** — pauses at HITL checkpoints for human approval
- **Auto mode** — runs end-to-end without pauses (for trusted, simple tasks)

## Project Context
If `.claude/context/` contains files from AIInitFramework, read them to understand the project:
- `ProjectOverview.md` — project description and architecture
- `ObjectModel.md` — custom objects and relationships
- `Automations.md` — triggers, flows, scheduled jobs
- `Integrations.md` — external system connections
- `SecurityModel.md` — profiles, permission sets, sharing rules
- `UIComponents.md` — Lightning pages, LWC components
- `BusinessProcesses.md` — business logic and processes

## Tool Agnostic
This framework works with any AI coding tool:
- **Claude Code** — reads this file automatically from `.claude/CLAUDE.md`
- **Cursor** — copy this content to `.cursorrules` or reference `.claude/` files
- **Other tools** — point to `.claude/` directory for framework content

## CLI Commands
Primary tool is `sf` (Salesforce CLI). All skills reference `sf` commands:
- `sf project deploy start` — deploy metadata
- `sf project retrieve start` — retrieve metadata
- `sf apex run test` — run Apex tests
- `sf org open` — open org in browser
