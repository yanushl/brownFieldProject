# Artifacts

> Ready-to-copy files that are NOT generated by sf-init-* skills.

## What's the difference?

| Folder | Purpose | How to use |
|--------|---------|------------|
| `skills/` | Analysis skills that generate context from YOUR project | Invoke skill, get generated output |
| `examples/rules/` | Sample output from skills | Reference only |
| `artifacts/` | Ready-to-use templates | Copy to your project and customize |

## Available Artifacts

| File | Purpose | Copy to |
|------|---------|---------|
| [local-deploy.md](local-deploy.md) | Deployment commands for local development | Platform-specific rules directory |

## How to Use

1. Copy the artifact to your platform's rules folder:
   - **Cursor:** `.cursor/rules/`
   - **Claude Code:** `.claude/rules/`
   - **Copilot:** `.github/instructions/` (rename to `*.instructions.md`)
   - **Windsurf:** Append to `.windsurfrules`
2. Customize placeholders (org alias, paths)
3. Commit to your repo

```bash
# Example for Cursor
cp artifacts/local-deploy.md /path/to/your/project/.cursor/rules/local-deploy.md

# Example for Claude Code
cp artifacts/local-deploy.md /path/to/your/project/.claude/rules/local-deploy.md
```

## Why Separate from Skills?

Skills ANALYZE your project and generate context based on what they find.

Artifacts are **generic templates** that don't need analysis — they work the same way for any Salesforce project. Teams just copy and customize.
