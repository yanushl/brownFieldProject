# Complete Project Analyzer

## Purpose

Orchestrate all analysis skills in the correct order to create a comprehensive project snapshot for AI agents.

**This runs ALL skills in sequence.** For individual areas, use specific skills directly.

## Output

Creates complete rules structure for detected platform(s).

### Platform Output

| Platform | Structure |
|----------|-----------|
| **Cursor** | `.cursor/rules/` with `globs` + `alwaysApply` frontmatter |
| **Claude Code** | `.claude/rules/` with `paths` frontmatter + `CLAUDE.md` |
| **Copilot** | `.github/instructions/*.instructions.md` with `applyTo` frontmatter |
| **Windsurf** | Single `.windsurfrules` file with all sections |

**Example for Cursor:**
```
.cursor/rules/
├── INDEX.md
├── project.md
├── business.md
├── data-model.md
├── architecture.md
├── apex.md
├── lwc.md
├── testing.md
├── integrations.md
├── security.md
└── devops.md
```

**Example for Claude Code:**
```
.claude/
├── CLAUDE.md          # Imports rules via @.claude/rules/...
└── rules/
    ├── project.md
    ├── apex.md
    └── ...
```

**Example for Copilot:**
```
.github/
├── copilot-instructions.md    # Repository-wide
└── instructions/
    ├── apex.instructions.md
    ├── lwc.instructions.md
    └── ...
```

## Execution Workflow

### Phase 0: Pre-flight Check

Before running skills, verify:

```
1. Check project structure:
   ├── Is sfdx-project.json present?
   ├── Is force-app/ directory present?
   └── Are there Apex classes/LWC components?

2. Check MCP availability (optional):
   ├── Salesforce MCP configured?
   ├── Chrome DevTools MCP available?
   └── Note which skills can run fully vs. partially

3. Detect target platform(s):
   ├── .cursor/rules/        → Cursor
   ├── .claude/rules/        → Claude Code
   ├── .github/instructions/ → Copilot
   ├── .windsurfrules        → Windsurf
   └── None found?           → Ask user
```

### Phase 1: Foundation (Context & Business)

**Run first — other skills may reference this context.**

| Order | Skill | Output | Dependencies |
|-------|-------|--------|--------------|
| 1 | `sf-init-context` | project.md | External docs, README |
| 2 | `sf-init-business` | business.md | Confluence, user input |

**Checkpoint:** Confirm project overview is accurate before continuing.

### Phase 2: Data & Architecture

**Run second — code analysis skills reference these.**

| Order | Skill | Output | Dependencies |
|-------|-------|--------|--------------|
| 3 | `sf-init-data-model` | data-model.md | Salesforce MCP or local metadata |
| 4 | `sf-init-architecture` | architecture.md | Local code analysis |

### Phase 3: Code Patterns

**Run third — detailed code analysis.**

| Order | Skill | Output | Dependencies |
|-------|-------|--------|--------------|
| 5 | `sf-init-apex` | apex.md | force-app/**/classes/ |
| 6 | `sf-init-lwc` | lwc.md | force-app/**/lwc/ |
| 7 | `sf-init-testing` | testing.md | Test classes, Jest tests |

### Phase 4: Infrastructure

**Run last — cross-cutting concerns.**

| Order | Skill | Output | Dependencies |
|-------|-------|--------|--------------|
| 8 | `sf-init-integrations` | integrations.md | Named credentials, callout classes |
| 9 | `sf-init-security` | security.md | Permission sets, sharing rules |
| 10 | `sf-init-devops` | devops.md | CI/CD configs, scripts |

### Phase 5: Generate INDEX.md

After all skills complete, generate/update INDEX.md:

```markdown
---
description: Navigation index for AI agents. Read this first.
globs: ["**/*"]
alwaysApply: true
---

# Project Context Index

> Navigation guide for AI agents. Read this file first.

## Quick Reference

| Topic | File | Status |

| Project overview | [project.md](project.md) | ✅ |

| Data model | [data-model.md](data-model.md) | ✅ |
| Architecture | [architecture.md](architecture.md) | ✅ |
| Apex patterns | [apex.md](apex.md) | ✅ |
| LWC patterns | [lwc.md](lwc.md) | ✅ |

3. **Know patterns**: `architecture.md` + `apex.md` — how code is structured
4. **For your task**: Read the relevant guide

---
```

## Handling Flags

### --skip

Skip specific skills:

```
User: /sf-init-all --skip lwc,integrations

Execution:
1. ✅ sf-init-context
2. ✅ sf-init-business
3. ✅ sf-init-data-model
4. ✅ sf-init-architecture
5. ✅ sf-init-apex
6. ⏭️ sf-init-lwc (SKIPPED)
7. ✅ sf-init-testing
8. ⏭️ sf-init-integrations (SKIPPED)
9. ✅ sf-init-security
10. ✅ sf-init-devops
```

### --only

Run only specific skills:

```
User: /sf-init-all --only apex,testing

Execution:
1. ⏭️ sf-init-context (SKIPPED)
...
5. ✅ sf-init-apex
...
7. ✅ sf-init-testing
...
```

### --refresh

Re-run all skills, overwriting existing files:

```
User: /sf-init-all --refresh
```

Default behavior: Skip skills if output file already exists and is recent (< 7 days).

```

1. Continue with remaining skills (default)

```

## Platform-Specific Output

### For Cursor

Files in `.cursor/rules/` with YAML frontmatter:

```markdown
---
description: [skill-specific]
globs: ["**/*.cls", "**/*.trigger"]
alwaysApply: false
---
```

### For Claude Code

Files in `.claude/rules/` with `paths` frontmatter:

```markdown
---
paths:
  - "**/*.cls"
  - "**/*.trigger"
---
```

Plus `CLAUDE.md` in project root that imports rules:

```markdown
# Project Instructions

@.claude/rules/project.md
@.claude/rules/apex.md
@.claude/rules/lwc.md
```

### For GitHub Copilot

Files in `.github/instructions/` with `applyTo` frontmatter:

```markdown
---
applyTo: "**/*.cls,**/*.trigger"
---
```

Plus `copilot-instructions.md` for repository-wide instructions.

### For Windsurf

Single `.windsurfrules` file with all sections concatenated:

```markdown
# Project Instructions

## Project Overview
[content from project.md]

## Apex Patterns
[content from apex.md]

...
```
